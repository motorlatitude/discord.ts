Classes DiscordClient Events GUILD_CREATE
Event

GUILD_CREATE

Event is emitted if the bot joins a guild or lazy loads guilds after connecting.

Declaration

event: "GUILD_CREATE"

Parameters

Guild: Guild

A Guild instance containing the newly joined guild.


Information
The created Guild Instance will be populated using available information from discords GUILD_CREATE event. When initially connecting, the gateway will only send offline members if a guild has less than the set large_threshold members. Additional members will have to be explicitly requested.

Discussion

This event is emitted if the bot joins a guild. The bot joins a guild in multiple scenarios as defined in the discord documentation;

Guild Create Event Scenarios
This event can be sent in three different scenarios:
  1. When a user is initially connecting, to lazily load and backfill information for all unavailable guilds sent in the Ready event.
  2. When a Guild becomes available again to the client.
  3. When the current user joins a new Guild.

The client will also emit "GUILD_CREATE" event in each of these cases, you should handle separating these event types yourself, based on your project scope.
