DiscordClient

An instance that controls events and initiates a connection with the discord gateway server.

Declaration

class DiscordClient extends events.EventEmitter

Overview

Every application wanting to utilise Discord's API gateway, will require to use a DiscordClient instance to keep track of Channels, Guilds, Users and more. The DiscordClient instances will issue events as they come in from the gateway after manipulating its internal components.

The DiscordClient instance can also be used to access discords REST API separately, without, using the gateway connection and can be accessed through the DiscordAPIManager property.

Gateway WebSocket Connection

The DiscordClient instance is written around event-driven architecture and extends the pre-existing EventEmitter class that is part of the events node module and will hence inherit all relevant properties and methods, depending on your node version. The instance method of particular use here is the on(event: listener) method and should be used to listen for relevant events that this library will emit.

DiscordTS will take over the role of caching and parsing incoming events and updating relevant internal components, keeping its state up-to-date. Projects should first see if cached values are available before sending API requests through the DiscordAPIManager so to avoid excessive or unnecessary API calls. Any API calls made will update internal components and are hence also cached.
