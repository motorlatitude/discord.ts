Classes DiscordClient Events MESSAGE_CREATE
Event

MESSAGE_CREATE

Event is emitted when a message is sent in a channel.

Declaration

event: "MESSAGE_CREATE"

Parameters

Message: Message

A new Message instance for the created message

Channel: TextChannel | DirectMessageChannel

The channel instance describing the channel in which the message was created

Author: User

A User instance for the person sending the message

Guild?: Guild

The Guild instance in which the TextChannel instance resides

GuildMember?: GuildMember

The GuildMember instance for the author

Discussion

This event is emitted when a new message is sent in a channel that the bot user is a member of.

Information
This event will also emit when the bot user itself send a message. This can be used to compare the Nonce of the sent message to make sure that the message has been sent successfully.

It should also be noted the author parameter will always be a User instance, however can hold limited information when the created message originates from a webhook as stated in Discord's API documentation;

Message Author Field
The author object follows the structure of the user object, but is only a valid user in the case where the message is generated by a user or bot user. If the message is generated by a webhook, the author object corresponds to the webhook's id, username, and avatar. You can tell if a message is generated by a webhook by checking for the webhook_id on the message object.

The Guild and GUILD_MEMBER parameters will only be populated if discords MESSAGE_CREATE event was sent from a TextChannel within a valid Guild instance and will otherwise be undefined. If the message was sent outwith a guild the Channel parameter will be a DirectMessageInstance.
