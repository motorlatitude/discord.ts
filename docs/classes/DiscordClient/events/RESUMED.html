Classes DiscordClient Events RESUMED
Event

RESUMED

Event is emitted once the connection is resumed.

Declaration

event: "RESUMED"

Parameters

This event does not return any values

Discussion

This event is emitted after the connection to the Gateway was lost and resumed. This event denotes the end of incoming missed payloads.

Your project should be aware of the possibility of incoming payloads after a DISCONNECT event. These payloads will follow the same format as any normal event and will be replayed in order. The RESUMED event will be emitted once all payloads have been received and the connection is open again.

The library will automatically try to reconnect and resume the connection in most cases, in cases where this is not possible or the resume request is refused by discord, the library will automatically attempt to go through the full connection flow and create a new connection after waiting for 5 seconds.
